# 00. Ruta general del curso

## Objetivo
Construir una aplicación Laravel completa de práctica con un dominio sencillo de **academia**, inspirada en la organización técnica de proyectos como `backend-EAC`, `eportfolio` y `marcapersonal`, pero con un contenido pensado para aprender y practicar examen.

## Qué se va a trabajar

1. Entorno en Debian
2. Git y GitHub
3. Composer, npm y Laravel Installer
4. Crear un proyecto Laravel desde cero
5. Estructura de carpetas de Laravel
6. Blade, layouts y componentes
7. Rutas web y rutas API
8. Controladores, requests, resources y services
9. Modelos, migraciones y relaciones Eloquent
10. Seeders y factories
11. Breeze y autenticación web
12. Sanctum y autenticación API
13. Swagger / L5 Swagger
14. Tests Feature y Unit
15. Estrategia de desarrollo en examen

## Filosofía de trabajo
Este repositorio tiene dos capas:

- **explicación**: para entender qué hace cada pieza
- **ejemplo mínimo funcional**: para tener una referencia clara

## Dominio de práctica
La aplicación gira en torno a:

- Centro
- Usuario
- Perfil
- Aula y horario
- Curso, lección y tarea
- Alumno y taller
- Comentario, etiqueta, foto y categoría

## Qué relaciones practicarás

- 1 a 1: 2 veces
- 1 a muchos: varias veces
- muchos a muchos: 2 veces
- hasManyThrough: 2 veces
- polimórficas 1 a 1: 2 veces
- polimórficas 1 a muchos: 2 veces
- polimórficas muchos a muchos: 2 veces
- autorrelaciones: 2 veces

## Orden recomendado

1. Instalación
2. Git/GitHub
3. Crear proyecto Laravel
4. Breeze y Blade base
5. Modelos y migraciones
6. Relaciones
7. Seeders/factories
8. CRUD web
9. API versionada
10. Sanctum
11. Swagger
12. Tests
13. Flujo de examen
